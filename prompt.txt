#Catalog
$ 
카탈로그시스템을 만들려고 해. 아래는 개략적인 나의 요구사항인데, 적절히 보완해 줘.
목적 : 회사 임직원들이 In-house로 개발한 SW자산을 쉽게 찾고, 활용할 수 있도록 하고, 또한 서로 SW재사용자산을 공유하기 위함이야.
기능요구사항 : 아래와 같은데, 적절히 추가해 줘도 좋아.
카탈로그를 통해 Item을 소개하고, 사용 신청을 받고 사용이력을 보여 주고, 카탈로그의 등록, 수정, 버전관리 기능을 제공 해줘. 
등록/수정은 자산오너가 하는데, Admin이 최종 승인을 해 줘야 반영이 돼.
카탈로그 등록시 주요 항목은 Description(Overview, Key Featuer, Prerequisite, Usage Guideline, Contact Point, Author, Githup URL 등), Categroy, 수정이력, 참고자료(첨부파일이나 URL 추가 가능)
카탈로그를 카드형식으로 보여 줄때는 하단에 활용횟수(이걸 클릭하면 활용처 목록 나와야 하지), 활용효과금액 합계이 나와야 해. 조회수, 좋아요도 나와야겠다.

카탈로그 사용후에는 사용후기도 작성 가능 해야 해.

월별, 분기별, 연도별 등 특정 주기별 실적과 추이도 볼 수 있어야 해.
관심 있는 카탈로그를 등록하면 변동사항을 통지 받을 수 있고, 필요하면 QnA도 할 수 있어.

현재는 회사내 in-house로 개발된 sw자산을 카탈로그에 담을 거야. 외부 판매용은 아니고. 나중에는 회사에서 구매한 외부 상품, 회사에서 판매용으로 만든 sw상품도 카탈로그에 포함 할 것 이지만, 판매나 구매용은 아니야. 사용 신청은 할 수 있어
qna와 자산 추천기능도 필요한데, ai가 대응토록 하려고 해. qna는 카탈로그(sw자산)에 대한 것도 있고, sw자산 활용정책도 있어. 활용정책은 md 파일로 되어 있어.

CX 요구사항 : 사용자인터페이스는 ServiceNow의 Servcie Catalog 처럼 Modern한 Style로 해 줘.

기술 요구사항 : Next.js 사용해줘.
